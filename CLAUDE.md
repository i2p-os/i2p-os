## Project Overview

**i2p-os** is an "I2P OS for integrating all things anonymous/private" - a Quasar Framework (Vue 3) application for I2P network integration.

## Project Structure

The application is located in `i2p-os-facade-quasar-app/i2p-os/`. This is a monorepo with the main app in this subdirectory.

```
i2p-os-facade-quasar-app/i2p-os/
├── src/
│   ├── boot/          # Boot files (axios.ts, i18n.ts)
│   ├── components/    # Vue components
│   ├── css/           # SCSS styles (app.scss, quasar.variables.scss)
│   ├── i18n/          # Internationalization (en-US)
│   ├── layouts/       # Vue layouts (MainLayout.vue)
│   ├── pages/         # Route pages (IndexPage.vue, ErrorNotFound.vue)
│   ├── router/        # Vue Router config (routes.ts, index.ts)
│   ├── stores/        # Pinia stores
│   ├── App.vue        # Root component
│   └── env.d.ts       # TypeScript declarations
├── quasar.config.ts   # Quasar CLI configuration
├── eslint.config.js   # ESLint configuration (flat config)
├── tsconfig.json      # TypeScript config (extends .quasar/tsconfig.json)
└── pnpm-workspace.yaml # Workspace configuration
```

## Development Commands

All commands should be run from `i2p-os-facade-quasar-app/i2p-os/`:

```bash
# Init

nvm install 

# Install dependencies
yarn install

# Start development server (auto-opens browser)
yarn dev
# or
quasar dev

# Build for production
quasar build

# Lint code
yarn lint

# Format code
yarn format
```

## Technology Stack

- **Framework:** Quasar v2.16 (Vue 3.4+)
- **Build Tool:** Vite (via @quasar/app-vite)
- **Language:** TypeScript (strict mode)
- **State:** Pinia
- **HTTP:** Axios (configured in boot/axios.ts)
- **I18n:** Vue I18n (configured in boot/i18n.ts)
- **Package Manager:** Yarn (yarn.lock present; pnpm-workspace.yaml also exists)
- **Node Version:** v24.14.1 (see .nvmrc)

## Key Configuration

### Boot Files
Boot files run before app mount. Current boot files:
- `i18n` - Vue I18n configuration with en-US messages
- `axios` - Axios instance with baseURL 'https://api.example.com'

### Vue Router
- Mode: `hash` (configurable in quasar.config.ts)
- Routes defined in `src/router/routes.ts`

### TypeScript
The project uses strict TypeScript. `tsconfig.json` extends `./.quasar/tsconfig.json` which is auto-generated by Quasar.

### ESLint
Uses flat config format (`eslint.config.js`) with:
- @quasar/app-vite recommended config
- Vue flat/essential rules
- TypeScript recommended type-checked rules
- Prettier skip-formatting (formatting handled separately)

## Multi-Platform Targets

The app is configured for multiple platforms (quasar.config.ts):
- **SPA** (default)
- **SSR** - Production port 3000
- **PWA** - Workbox GenerateSW mode
- **Electron** - Bundler: packager, preload: electron-preload
- **Cordova**
- **Capacitor**
- **BEX** (Browser Extension)

## Important Notes

- **App.vue vs MainLayout:** The current `App.vue` has a custom I2P-themed dark layout that doesn't use `MainLayout.vue`. MainLayout exists but isn't currently the root layout.
- **Vite Plugins:** vite-plugin-checker runs type checking and ESLint during development
- **Development:** Dev server opens browser automatically (`devServer.open: true`)
